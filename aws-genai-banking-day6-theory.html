üîÑ Fine-tuning vs RAG

Choosing the Right Approach for Banking LLMs

Two Approaches to Custom LLMs

üéØ Fine-tuning

  • What: Retrain model on custom data
  • When: Need specialized behavior
  • Cost: High (training compute)
  • Time: Hours to days
  • Updates: Requires retraining

üìö RAG (Retrieval Augmented Generation)

  • What: Retrieve relevant context
  • When: Need current information
  • Cost: Low (inference only)
  • Time: Minutes to setup
  • Updates: Just update knowledge base

When to Use Each Approach

Banking regulations Q&A‚ùå Outdated quickly‚úÖ Always currentRAG

Customer service chatbot‚úÖ Brand voice‚úÖ Product infoBoth

RAG Best Practices

‚úÖ Do

  • Chunk Size: 500-1000 tokens optimal
  • Overlap: 10-20% between chunks
  • Metadata: Store source, date, category
  • Reranking: Use cross-encoder for top results
  • Caching: Cache frequent queries

‚ùå Don't

  • Too Large: >2000 token chunks lose context
  • No Metadata: Can't filter by source
  • Single Retrieval: Always get top-k (3-5)
  • Ignore Relevance: Check similarity scores
  • Stale Data: Update index regularly
‚ö†Ô∏è
Data Privacy: Banking documents contain sensitive data. Use encryption at rest, access controls, and audit logging. Consider on-premises vector DB for highly sensitive data.

Critical for Banking: Choosing the right instance type impacts cost, performance, and compliance. This guide helps you make informed decisions for production ML workloads.

Instance Selection Decision Tree

CPU vs GPU Selection Criteria

üñ•Ô∏è Use CPU Instances When:

  • Traditional ML: XGBoost, Random Forest, Linear models
  • Small Datasets: < 100K records
  • Tabular Data: Customer data, transactions
  • Inference: Real-time predictions with low latency
  • Cost-Sensitive: Budget constraints
üí∞
Cost: $0.05 - $0.50/hour

üöÄ Use GPU Instances When:

  • Deep Learning: Neural networks, transformers
  • Large Models: LLMs, computer vision
  • Image/Video: Check fraud detection, KYC
  • NLP Tasks: Document analysis, sentiment
  • Training Speed: Need faster iterations
‚ö†Ô∏è
Cost: $3 - $30/hour (10-100x CPU)

Cost Optimization Strategies

üí∞
Development Phase:
  • Use ml.t3.medium for endpoint testing (endpoints only)
  • Use ml.m5.large for training experiments
  • Delete endpoints when not in use
  • Use spot instances for training (70% savings)
  • Limit training time with early stopping
üöÄ
Production Phase:
  • Use auto-scaling for variable traffic
  • Consider serverless for unpredictable loads
  • Use multi-model endpoints to share resources
  • Monitor CloudWatch metrics for right-sizing
  • Use Savings Plans for 30-50% discount
‚ö†Ô∏è
Critical Instance Type Rules:
  • T-series (T2, T3) ONLY work for endpoints - They will FAIL for training jobs
  • M-series (M5, M4) work for both training and endpoints
  • Always test with small instances first before scaling up
  • GPU instances are 10-100x more expensive - use only when necessary
  • Serverless has cold start latency (5-10 seconds) - not for real-time critical apps

  • Maximum > 85%: Immediate action needed

Method 3: AWS Lambda Integration

‚ö° Lambda Function Example

import json
import boto3
import os

# Initialize client outside handler for reuse
sagemaker_runtime = boto3.client('sagemaker-runtime')
ENDPOINT_NAME = os.environ['SAGEMAKER_ENDPOINT_NAME']

def lambda_handler(event, context):
    """
    Lambda function to invoke SageMaker endpoint
    Triggered by API Gateway, S3, EventBridge, etc.
    """
    try:
        # Extract input from event
        if 'body' in event:
            # API Gateway event
            input_data = json.loads(event['body'])
        else:
            # Direct invocation
            input_data = event
        
        # Invoke SageMaker endpoint
        response = sagemaker_runtime.invoke_endpoint(
            EndpointName=ENDPOINT_NAME,
            ContentType='application/json',
            Body=json.dumps(input_data)
        )
        
        # Parse prediction
        result = json.loads(response['Body'].read().decode())
        
        # Return response
        return {
            'statusCode': 200,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*'  # CORS
            },
            'body': json.dumps({
                'prediction': result,
                'endpoint': ENDPOINT_NAME,
                'timestamp': context.request_id
            })
        }
        
    except Exception as e:
        print(f"Error: {str(e)}")
        return {
            'statusCode': 500,
            'body': json.dumps({'error': str(e)})
        }

# Required IAM permissions for Lambda execution role:
# - sagemaker:InvokeEndpoint
# - logs:CreateLogGroup, logs:CreateLogStream, logs:PutLogEvents

Error Handling & Retry Logic

üîÑ Robust Invocation with Retries

import boto3
import json
import time
from botocore.exceptions import ClientError

def invoke_endpoint_with_retry(endpoint_name, payload, max_retries=3):
    """
    Invoke SageMaker endpoint with exponential backoff retry logic
    """
    runtime = boto3.client('sagemaker-runtime', region_name='us-east-1')
    
    for attempt in range(max_retries):
        try:
            response = runtime.invoke_endpoint(
                EndpointName=endpoint_name,
                ContentType='application/json',
                Body=json.dumps(payload)
            )
            
            # Success - parse and return result
            result = json.loads(response['Body'].read().decode())
            return {
                'success': True,
                'prediction': result,
                'latency_ms': response['ResponseMetadata']['HTTPHeaders'].get('x-amzn-sagemaker-inference-time', 'N/A'),
                'attempt': attempt + 1
            }
            
        except ClientError as e:
            error_code = e.response['Error']['Code']
            error_message = e.response['Error']['Message']
            
            # Handle specific errors
            if error_code == 'ModelError':
                # Model inference error - don't retry
                return {
                    'success': False,
                    'error': 'Model inference failed',
                    'details': error_message
                }
            
            elif error_code == 'ValidationException':
                # Invalid input - don't retry
                return {
                    'success': False,
                    'error': 'Invalid input format',
                    'details': error_message
                }
            
            elif error_code in ['ThrottlingException', 'ServiceUnavailable']:
                # Transient errors - retry with backoff
                if attempt < max_retries - 1:
                    wait_time = (2 ** attempt) + (random.randint(0, 1000) / 1000)
                    print(f"Attempt {attempt + 1} failed. Retrying in {wait_time:.2f}s...")
                    time.sleep(wait_time)
                    continue
                else:
                    return {
                        'success': False,
                        'error': 'Max retries exceeded',
                        'details': error_message
                    }
            
            else:
                # Unknown error
                return {
                    'success': False,
                    'error': f'Unexpected error: {error_code}',
                    'details': error_message
                }
        
        except Exception as e:
            # Catch-all for unexpected errors
            return {
                'success': False,
                'error': 'Unexpected exception',
                'details': str(e)
            }
    
    return {
        'success': False,
        'error': 'Max retries exceeded'
    }

# Usage
result = invoke_endpoint_with_retry(
    endpoint_name='securebank-churn',
    payload={'age': 35, 'balance': 50000}
)

if result['success']:
    print(f"Prediction: {result['prediction']}")
    print(f"Latency: {result['latency_ms']}ms")
    print(f"Attempts: {result['attempt']}")
else:
    print(f"Error: {result['error']}")
    print(f"Details: {result['details']}")

Security Best Practices

üîí
Security Checklist for Endpoint Usage:
  • IAM Permissions: Use least-privilege principle - grant InvokeEndpoint only to specific endpoints
  • VPC Endpoints: Use VPC endpoints for private connectivity (no internet exposure)
  • Encryption: Data encrypted in transit (TLS 1.2+) and at rest
  • Input Validation: Validate and sanitize all input data before sending to endpoint
  • Rate Limiting: Implement rate limiting to prevent abuse and control costs
  • Logging: Enable CloudWatch Logs for audit trail and compliance
  • Monitoring: Set up CloudWatch alarms for error rates and latency spikes
  • Secrets Management: Use AWS Secrets Manager for API keys/credentials (not hardcoded)
  • Network Security: Use security groups to restrict access to endpoints
  • Data Privacy: Ensure PII/sensitive data is handled per compliance requirements (GDPR, PCI-DSS)
