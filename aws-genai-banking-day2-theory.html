CIDR Block Planning for SecureBank

üè¶ IP Address Strategy

VPC CIDR: 10.0.0.0/16 (65,536 IPs)

  • Public Subnets: 10.0.1.0/24, 10.0.4.0/24
  • Private App: 10.0.2.0/24, 10.0.5.0/24
  • Private DB: 10.0.3.0/24, 10.0.6.0/24
  • Management: 10.0.7.0/24, 10.0.8.0/24
  • Reserved: 10.0.9.0/24 - 10.0.255.0/24

Subnet Sizing Strategy:

  • /24 Subnets: 254 usable IPs each
  • Growth Planning: 50% current, 50% future
  • Multi-AZ: Identical subnets per AZ
  • Compliance: Separate subnets for audit
  • Disaster Recovery: Cross-region replication

VPC Components Deep Dive

üåê Internet Gateway

  • Purpose: Provides internet access to VPC
  • Attachment: One per VPC
  • Routing: 0.0.0.0/0 ‚Üí IGW
  • Security: Stateless, allows all traffic
  • Banking Use: Customer-facing services

üîÑ NAT Gateway

  • Purpose: Outbound internet for private subnets
  • Placement: Public subnet, one per AZ
  • Routing: Private route tables
  • Security: Outbound only, no inbound
  • Banking Use: Software updates, API calls

üìã Route Tables

  • Purpose: Control traffic routing
  • Types: Main, custom, local routes
  • Association: Subnet-level assignment
  • Priority: Most specific route wins
  • Banking Use: Network segmentation

Implementing Defense in Depth for Banking Security

Security Groups vs Network ACLs Comparison

SecureBank Defense in Depth Strategy

üè¶ Multi-Layer Security Architecture

Layer 1 - Network ACLs:

  • Public Subnet: Allow HTTP/HTTPS, deny database ports
  • App Subnet: Allow app ports from public, deny direct internet
  • DB Subnet: Allow database ports from app tier only
  • Management: Allow SSH from bastion, deny all else

Layer 2 - Security Groups:

  • ALB-SG: 80/443 from internet, 8080 to app tier
  • App-SG: 8080 from ALB-SG, 3306 to DB-SG
  • DB-SG: 3306 from App-SG only
  • Bastion-SG: 22 from office IP, 22 to private instances

Banking Security Group Examples

üåê Web Tier Security Group

üè¶ Application Tier Security Group

# App-SecurityGroup Inbound Rules: - Port 8080: ALB-SecurityGroup - Port 22: Bastion-SecurityGroup Outbound Rules: - Port 3306: DB-SecurityGroup - Port 443: 0.0.0.0/0 (API calls) - Port 80: 0.0.0.0/0 (updates) # Use Case: Banking application servers # Compliance: Restricted access, audit logging
